使用Hexo框架和Github Pages搭建静态博客

发表于 2022-08-10 更新于 2022-08-11

Hexo was originally created and maintained by Tommy Chen in 2012. Since then, it has helped thousands of people to build their dream website/blog.

序

&ensp;作为一名“Geek”，哪能没有自己的博客？这就好像一名司机必须要有车，而车需要花买，服务器也需要。但好在博客不一定要有自己的服务器。车，也不一定非要是自己的。

&ensp;2020年的夏天，Hexo框架偶然进入我的视野，这是一个基于Node.js构建的静态博客框架，该框架以Markdown作为博文源码，根据自定义的博客主题自动生成html页面。配合Github pages自动部署，甚至可以做到零成本搭建博客。

&ensp;时隔两年，恍惚中，我竟再次回到了这里，只愿记录下自己搭建博客的一点心得，为大家献上一篇微不足道的教程。

一切开始之前的准备

&ensp;正如前文所述，Hexo基于Node.js构建，考虑到Node.js开发环境在Windows系统上的种种不便利因素，我选择了WSL2（Debian），如果你坚持要在Windows上完成搭建，这篇博文依然适用，不过你可能需要格外找方法在Windows上搭建下述环境（因为我懒）。

安装Node.js，npm，以及hexo模块

&ensp;首先我们需要安装Node.js运行时及其包管理器npm。

Node.js发布于2009年5月，由Ryan Dahl开发，是一个基于Chrome V8引擎的JavaScript运行环境，使用了一个事件驱动、非阻塞式I/O模型，让JavaScript 运行在服务端的开发平台，它让JavaScript成为与PHP、Python、Perl、Ruby等服务端语言平起平坐的脚本语言
百度百科

&ensp;在Debian上可以使用命令apt install nodejs来安装nodejs（需要管理员权限），其余发行版请使用相应的包管理器并寻找对应的包名，下同。
&ensp;Hexo框架并不是Node.js的一部分，它被封装为Node.js的一个第三方模块，需要使用Node.js的包管理器（npm）进行安装。在Debian上可以使用命令apt install npm进行安装。
&ensp;接着我们需要使用npm安装Hexo框架，使用指令npm install -g hexo-cli即可（需要管理员权限），其中，-g代表全局安装。

搭建Github Pages

&ensp;由于没有专用的云服务器用于运行博客，我们需要寻找一家可靠的托管方，比如Github。

&ensp;首先你需要登入你的Github账号，并创建一个新的Repository，其命名格式需为username.gitHub.io（比如我的，iaksh.github.io）。符合该命名格式的Repository会自动进行Github Pages构建。
当然，截至目前，这里可能只有一个孤零零的README.md。

创建你的Hexo博客

&ensp;找一个你喜欢的地方兴建文件夹，然后在终端中cd进哪个文件夹并使用命令hexo init，一个hexo博客工程就完成了。
此时你可以使用hexo server（或简写为hexo s)试运行博客，使用浏览器访问localhots:4000即可预览。
使用hexo new name（hexo n name）即可开始一篇新的博文，此命令将会在工程根目录的source文件夹中生成对应名称的markdown源文件，该markdown源文件即为你的博文源码。
其他的命令还有hexo generate（hexo g），hexo deploy（hexo d）。前者能将博客生成为一份完整的html+css+js文件，后者自动将其按照预设配置进行部署

修改你的博客配置

&ensp;工程根目录中有一配置文件名为“_config.yml”，这是hexo最顶层的配置文件，在这里你可以指定网站的一些基本内容。此外，你可以将第三方主题包放在工程根目录的themes文件夹中，然后修改主题包内的“_config.yml”。
另：由于各个主题配置文件不尽相同，具体请见其中的注释。

将你的Hexo博客推送至Github并自动部署

&ensp;在推送博客至github之前，你需要在工程根目录启用hexo的git部署模块，命令为npm install hexo-deployer-git –save
&ensp;完成上述操作后，还需修改工程根目录下的“_config.yml”中的deploy项目，将其中的type改为git，添加repo子项以指定git仓库，添加branch子项以指定上传分支。
&ensp;在这之后，使用指令hexo g && hexo d即可自动完成博客的生成与部署。

迟来的再会

其实我从2020年就开始写博文了，但受种种干扰，更新总是断断续续，最后神隐。

考虑到学校组织要求，以及我个人仅存的那一丁点追求，我决定从现在开始，重启博客。
我会花点时间把我近期的经历与经验，还有心得，总结成文，然后再把从前的博文再写一遍。

奥，对了，由于我不是社交恐怖分子，写文说话大概都会很意识流，要是觉得难受就用你的大拳头揍翻你面前正在显示这行字的屏幕吧（

这里大概会写一些各种IT相关的，以及游戏，还有一些可能的碎碎念。

我从前的博客在about页里写了“既然你能看见这些那大概是我认识的人，那问候什么的就免了吧”
虽然现在依然懒得写（是的，懒得写）问候之类的，但是这个博客也许不再是我的树洞了，我可能会在各种地方宣传它，以后可能会加上评论区，可能接少量广告，可能刚写了一句话又觉得不妥，然后删删改改一个小时，可能写了文章以后又害怕被谁看见所以突然删掉，可能，有很多可能…
不过还是以后再说吧。
